## 可能需要调整的问题

### 1. 模型太大或太小

**调整方法**：修改 `src/systems/ModelLoader.js` 中的 `scale` 值

```javascript
1: { 
  name: '迅猛龙', 
  url: '/models/raptor.fbx',
  fallback: false,
  scale: 0.01,  // ← 调整这个值
  color: 0x00ff00
}
```

**调整建议**：
- 模型太大 → 减小数值（例如：`0.01` → `0.005`）
- 模型太小 → 增大数值（例如：`0.01` → `0.02`）
- 建议范围：`0.005` - `0.05`

### 2. 模型方向不对

如果模型面朝错误方向，需要在加载后旋转。

**修改位置**：`src/systems/ModelLoader.js` 的 `loadDinoModel` 方法

在这段代码后添加：
```javascript
// 设置模型属性
model.scale.setScalar(config.scale);

// 添加旋转调整（如果需要）
model.rotation.y = Math.PI;  // 旋转180度
// 或
model.rotation.y = Math.PI / 2;  // 旋转90度
```

### 3. 模型位置偏移

如果模型的中心点不在正确位置（例如悬空或陷入地面）：

**调整Y轴偏移**：
```javascript
model.position.y = 0.5;  // 向上调整
// 或
model.position.y = -0.5;  // 向下调整
```

### 4. 模型材质/颜色问题

FBX模型应该自带材质和纹理。如果颜色不对：

```javascript
model.traverse((child) => {
  if (child.isMesh) {
    child.castShadow = true;
    child.receiveShadow = true;
    
    // 如果需要强制改变颜色
    if (child.material) {
      child.material.color.setHex(config.color);
    }
  }
});
```

## 常见问题

### Q: 模型加载失败？
**检查清单**：
- [ ] 文件确实在 `public/models/` 文件夹中
- [ ] 文件名拼写完全一致（注意大小写：`Trex.fbx`）
- [ ] FBX文件没有损坏（可以用Blender等软件打开测试）

### Q: 模型看不见？
**可能原因**：
1. 模型太小 → 增大 `scale` 值
2. 模型在地下 → 调整 `position.y`
3. 材质透明 → 检查模型材质设置

### Q: 模型动画不播放？
FBX模型的动画会自动检测并播放。查看控制台是否显示：
```
🎬 加载了 X 个动画
🎬 动画已启动: Walk
```

### Q: 想替换第5个恐龙？
如果想用雷龙（Apatosaurus）替换副栉龙：

```javascript
5: { 
  name: '雷龙', 
  url: '/models/Apatosaurus.fbx',  // 替换这里
  fallback: false,
  scale: 0.01,
  color: 0x8800ff
}
```

同时更新 `src/systems/EvolutionSystem.js` 中的名称。

```javascript
// 小型恐龙（迅猛龙）
scale: 0.01 - 0.015

// 中型恐龙（三角龙、剑龙）
scale: 0.015 - 0.02

// 大型恐龙（霸王龙、副栉龙）
scale: 0.02 - 0.03
```

## 性能优化提示

如果游戏运行卡顿：
1. 检查FBX模型的多边形数量（推荐 < 20000面）
2. 压缩或降低纹理分辨率
3. 关闭不必要的动画
